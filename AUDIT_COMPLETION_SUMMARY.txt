CRITICAL ISSUE: DETAILED ANALYSIS
════════════════════════════════════════════════════════════════════════════

VULNERABILITY: Account Lockout Bypass via Method Switching

Current Architecture (VULNERABLE):
┌──────────────────────────────────────────────────────────┐
│ Each 2FA Method Has Its Own Lockout Counter             │
├──────────────────────────────────────────────────────────┤
│ TOTP Method:       totp_locked_alice                    │
│ Backup Code:       backup_code_locked_1                │
│ Email OTP:         email_otp_locked_1                  │
│                                                          │
│ Attacker Flow:                                           │
│ 1. Try TOTP 3x → Locked on TOTP                         │
│ 2. Switch to Backup Code → NOT locked!                 │
│ 3. Try Backup Code 3x → Locked on Backup Code          │
│ 4. Switch to Email OTP → NOT locked!                   │
│ 5. Try Email OTP 3x → Locked on Email OTP              │
│ Total: 9 attempts (should be 3!)                        │
└──────────────────────────────────────────────────────────┘

Recommended Architecture (FIXED):
┌──────────────────────────────────────────────────────────┐
│ Single Account-Level Lockout for ALL Methods           │
├──────────────────────────────────────────────────────────┤
│ Unified Counter:   2fa_fail_count_1                    │
│ Unified Lock:      account_2fa_locked_1                │
│                                                          │
│ User Flow:                                               │
│ 1. Try TOTP 3x → Account locked (all methods)          │
│ 2. Try Backup Code → 429 Locked (blocked!)             │
│ 3. Try Email OTP → 429 Locked (blocked!)               │
│ Total: 3 attempts (correct!)                            │
└──────────────────────────────────────────────────────────┘

Implementation: See LOCKOUT_FIX_CODE.md
